Симметрич.шифр-е - один и тот же ключ для шифр. и расшифр.
Ассиметричное шифр-е - два ключа для шифровки и расшифр-ки. Один за зашифровывания (открытый),
		       другой для расши-ки (закрытый)

В большинстве алгоритмов симметр.шифр-я размер зашифр.текста == размер открытого текста
? Современный размер ключа - 128-256 бит для алгоритмов симметричного шифрования

Ассим-е шифрование
Ассиметр-е шифр-е - открытый ключ передается по открытому незащищ.каналу и использ-ся для
шифрования и проверки ЭЦП. Для расш-ки сообщ-я и генерации ЭЦП исп-ся секретный ключ.

Схема реализации:
	Два собеседника: Алиса и Боб. Алиса хочет -> передать Бобу инфу
	- Генерация ключевой пары
	  Боб выбирает алгоритм и пару (открыт + закрыт.ключ.), посылает открытый ключ Алисе по
	  открыт.каналу
	- Шифрование и передача сообщ-я
	  Алиса шифрует инфу с исп-ем открыт.ключа Боба. Передает полученный шифротекст
	- Расшифр-е сообщ-я
	  Боб с пом.закрыт.ключа расшифр-ет шифротекст

Электронная подпись
Позволяет подтвердить авторство электронного документа.

Виды ЭП:
	- простая (связка логин-пароль или СМС-код)
	- неквалифицированная - подтвержд.личность владельца и защищ.док.от измен-й,
	  но не содержит спец.ср-в шифрования, серт-х ФСБ России. Подход.для внут. док-оборота.
	- квалифицированная ЭП - то, что чаще всего называют ЭЦП. Выдается только центрами.
	  Док.подписанный квал.ЭП обладает юр.силой.

Серт.ЭП (серт.ключа проверки ЭП, серт.ЭП, серт.ключа ЭЦП) - док.кот.подтвержд.принадлеж-ть
подписи владельцу. Выдается только Уд.Центрами.
Сертификат открытого ключа (сертицикат ЭП, сертификат ключа подписи) - электрон. или бум. документ,
содержащий ОТКРЫТЫЙ ключ, инфу о владельце, область применения ключа, подписанный УЦ (удост.центром)

Сожерждание серт.ЭП:
	- открытый ключ - ключ проверки ЭП, содерж-щий код, с пом.кот.можно проверить подлинность ЭП
	  (ФИО, СНИЛС и др.). По откр.ключу получатель ЭД может убедиться, что док.не изменялся, ЭП действует
	- закрытый ключ - ключ для расшифр.док-а
	- номер серт-та ЭП
	- реквизиты владельца
	- ключ проверки ЭП для идентификации!!!???
	- стандарты сертификата
	- наименование и адрес УЦ
	- дату выдачи и срок действия серт-та (обычно 12 мес)

Ключ ЭП - уникальная послед-ть символов для создания ЭП (?сам закрытый ключ?)
Ключ проверки ЭП - уник.послед-ть сиволов, однозначно связ-я с ключом ЭП и предназнач-я для
проверки подлин-сти ЭП (?сам открытый ключ?)

КриптоПро
Компания, разрабатывающая одноименную линейку криптографических утилит - крипторовайдеров.
Используются в др.программах для генерации ЭП, работы с сертификатами, организации структуры PKI и др.

КриптоПро CSP - исп-ся в прог-х для сдачи налоговой отч-сти, в клиент-банках и пр.

Для подписи ЭД и обмена бланками необходим криптовровайдер, который шифрует и защищает инфу, хранит секретные
ключи.

cptools (CryptoPro tools) - графическая программа для создания и проверки ЭЦП, управления контейнерами и серт-
ми криптопровайдеров КриптоПро.

SSL
Secure Sockets Layer (SSL) - уровень защищенных сокетов - криптографический протокол. Использует ассиметр.
криптографию для аутентификации ключей, симметричое шифрование для сохр-я конф-сти

Протокол широко применяется для обмена мгн.сообщ-ми и передачи голоса через IP (Voice over IP) - электронная
почта, интернет факс.

SSL - уязвимый протокол. Уступил в пользу TLS. Но название прижилось, поэтому сегодня под SSL часто
подразумевают TLS
SSL изначально разработан для добавления протокола HTTPS в браузер.

Протокол обеспечивает:
	- аутентификацию
	- шифрование
SSL предоставляет безопасный канал.

SSL использует среду между несколькими слоями.
Размещается между двумя протоколами:
	- протоколом, кот.использует программа-клиент (HTTP, FTP, LDAP, TELNET и т.д.) и
	- протоколом транспортного уровня TCP/IP.

Защищает данные, выступая в роли фильтра для обех сторон и передает их далее на транспортный уровень.

SSL использует сертификаты для проверки принадлежности открытого ключа его реальному владельцу.
Способы получения сертификата?
	- использовать сертификат, выданный УЦ (CA - Certification Authority)
	- использовать самоподписанный сертификат
	- использовать "пустой" сертификат

Механизмы образования ключа для текущей сессии в SSL/TLS (алгоритмы):
	- RSA
	- Fixed Diffie-Hellman
	- Ephemeral Diffie-Hellman
	- Anonymous Diffie-Hellman


OpenSSL - криптографическая библиотека с открытым исх.кодом
Поддерживает почти все низкоуровневые аргоритмы хэширования, шифрования и ЭП.
Реализуеет большинство популярных криптографических стандартов, в т.ч. позволяет создавать ключи RSA, DH,
DSA, сертификаты X.509, подписывать их, формировать CSR и CRT
